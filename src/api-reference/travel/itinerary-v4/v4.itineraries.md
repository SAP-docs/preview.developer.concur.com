# Itineraries v4

The Itineraries API provides clients and authorized partners access to historical data.

## <a name="limitations"></a>Limitations

* This API is only available to clients and partners who have been granted access by SAP Concur. Access to documentation does not provide access to the API. This API is only available in US and EMEA data centers.
* This API provides historical data for the last _6 months_. It is meant to allow authorized clients and partners to pull the initial data they may need to get setup.
* This API has an aggressive rate limit of _10 requests per minute_. This API should not be used to poll for updates, only for an initial data fetch. Events will be fired for all changes going forward.

## <a name="prior-versions"></a>Prior Versions

* Itinerary v1 documentation is available [here](https://developer.concur.com/api-reference/travel/itinerary/itinerary.html).

## Scope Usage <a name="scope-usage"></a>

|Name|Description|Endpoint|
|---|---|---|
`travel.itinerary.read`|Allows user to read travel itinerary data.|GET

## Dependencies <a name="dependencies"></a>

None.

## Access Token Usage <a name="access-token-usage"></a>

This API supports only company level access tokens.

### URI

```shell
GET https://{region}.api.concursolutions.com/travel/v4/trips
```

### Parameters

|Parameter|Required|Type|Description
| ----- | ----- | ----- | -----
schema | No| string | Supported values - id
endDateFrom|No|string |Start date of `endDate` for the query list, Dateformat - `yyyy-mm-dd`, e.g.`2006-01-02`.  Date cannot be older than 1 year from now. Default value is NOW.
endDateTo| No|string | End date of `endDate` for the query list, Dateformat - `yyyy-mm-dd`, e.g.`2006-01-02`. Date cannot be older than `endDateFrom`. Results include trips with this date.
startAfter| No|string |A specific Id supporting pagination of results. Get results after this Id if provided. Response include NEXT and PREV page URI with this Id.


#### Response

Response includes 10 trips at most and NEXT or PREV operations in JSON format for each request

## Schema <a name="schema"></a>

Data|Operations
---|---
`data`|`array`|[`Itinerary Element`](#schema-itin)|A parent element that contains all Itinerary elements.
`operations`|`array`|[`Operations](#schema-operations)|A parent element that contains all pagination operations.

### <a name="schema-itin">Itinerary Element

Id|Href
---|---
`id`|`string`|-|Unique id for an itinerary
`href`|`string`|-|Direct Link to itinerary

### <a name="schema-operations"></a>Operation Element
`rel`|`string`|`next`/`prev`|-|Operation type
`href`|`string`|-|Operation URL
